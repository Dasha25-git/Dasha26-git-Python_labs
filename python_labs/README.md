# Лабораторная работа №2

## Тема

Контейнер объектов для предметной области "Библиотека / Книги".

## Цель работы

Реализовать коллекцию объектов `Book` с использованием принципов объектно-ориентированного программирования: инкапсуляции, типизации, валидации данных и методов управления коллекцией.

## Описание классов

### `Book`

Класс книги, перенесённый из ЛР-1 без изменения бизнес-логики. Хранит основные данные о книге и предоставляет методы для изменения её состояния.

Основные возможности:

- валидация входных данных при создании объекта;
- свойства для безопасного чтения данных;
- setter для цены с валидацией;
- бизнес-методы `mark_borrowed()`, `mark_returned()`, `mark_damaged()`, `repair()`, `mark_lost()`;
- метод `apply_discount()` для применения скидки;
- сравнение книг через `__eq__()` по названию, автору и году издания.

### `Library`

Класс-контейнер для хранения объектов `Book` на основе списка `list[Book]`.

Поддерживает:

- добавление книги `add(book)`;
- удаление книги `remove(book)`;
- удаление по индексу `remove_at(index)`;
- получение всех книг `get_all()`;
- поиск по названию `find_by_title(title)`;
- поиск по автору `find_by_author(author)`;
- итерацию по библиотеке;
- получение длины через `len(library)`;
- доступ по индексу через `library[index]`;
- сортировку по цене `sort_by_price()`;
- сортировку по году `sort_by_year()`;
- фильтрацию доступных книг `get_available()`;
- фильтрацию дорогих книг `get_expensive(min_price)`.

## Особенности реализации

- для хранения используется только `list`;
- нельзя добавить объект, который не является экземпляром `Book`;
- нельзя добавить дубликат книги;
- при ошибках выбрасываются исключения;
- методы фильтрации возвращают новую коллекцию `Library`, а не исходный объект.

## Как запустить демонстрацию

Из корня проекта выполните:

```bash

```
